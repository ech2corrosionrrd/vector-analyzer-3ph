# VectorAnalyzer 3Ph - Інструкція зі встановлення

Професійний вебдодаток для візуалізації та аналізу векторних діаграм трифазних мереж.

## 1. Початкові вимоги
Для запуску додатка на комп'ютері повинен бути встановлений **Node.js** (версія 18 або новіша).
Завантажити можна тут: [nodejs.org](https://nodejs.org/)

## 2. Встановлення та запуск на комп'ютері

1. **Завантажте проєкт** (якщо ви отримали архів, розархівуйте його в окрему папку).
2. **Відкрийте термінал** (PowerShell або CMD) у папці з проєктом.
3. **Встановіть залежності**:
   ```bash
   npm install
   ```
   Якщо `npm` повідомляє про **конфлікт peer-залежностей**, спробуйте:
   ```bash
   npm install --legacy-peer-deps
   ```
4. **Запустіть додаток у режимі розробки**:
   ```bash
   npm run dev
   ```
5. Після запуску термінал покаже посилання (зазвичай `http://localhost:5173`). Відкрийте його у браузері.

**Якщо `npm` «не знайдено»:** встановіть [Node.js LTS](https://nodejs.org/), перезапустіть ПК або принаймні закрийте всі вікна CMD/PowerShell і запустіть `start.bat` знову. Скрипт `start.bat` також шукає `npm` у типових папках (`Program Files`, Volta, Scoop тощо), якщо PATH ще не оновився.

**Шлях до проєкту латиницею (рекомендовано для CMD):** якщо у назві папки є кирилиця, `cmd.exe` інколи помиляється зі шляхом і змінними в батниках. Найпростіше **перейменувати** кореневу папку, наприклад на `vector-analyzer-3ph` або транслітом `Pobudova-vektornoii-diagramy`, і відкрити проєкт уже з нового шляху.

**Якщо «The system cannot find the path specified»** одразу після відкриття CMD: у властивостях ярлика CMD поле **Start in** / «Робоча папка» має бути порожнім або вказувати на **існуючий** каталог.

Файл **`start.bat`** навмисно лише **латиницею (ASCII)**, без кирилиці — так надійніше для Windows.

**Помилка `Cannot find module ... node_modules\npm\bin\...`:** у каталозі проєкту з’явилась зайва або зламана папка `node_modules\npm` (її немає серед залежностей у `package.json`). Видаліть її вручну або запустіть оновлений **`start.bat`** — він прибирає її та викликає **глобальний** `npm.cmd` з установки Node.js (`Program Files\nodejs`), а не з `node_modules`.

## 3. Корисні команди (розробка)

| Команда | Призначення |
|--------|-------------|
| `npm run dev` | Локальний сервер розробки (зазвичай `http://localhost:5173`) |
| `npm run build` | Збірка для продакшену в каталог `dist/` |
| `npm run preview` | Перегляд зібраної збірки локально |
| `npm run lint` | Перевірка коду ESLint |

## 4. Встановлення як додатка (PWA)

Додаток підтримує технологію **PWA**, що дозволяє встановити його на смартфон або комп'ютер як окрему програму, яка працюватиме навіть без інтернету.

Є **два окремі маніфести** (дві «програми» в системі встановлення):

| Варіант | Типовий шлях на GitHub Pages* | Призначення |
|--------|-------------------------------|-------------|
| Основний (ПК) | `https://<користувач>.github.io/<репозиторій>/` | Робоча сторінка за замовчуванням |
| Мобільний | `https://<користувач>.github.io/<репозиторій>/mobile/` | Окрема іконка / scope для телефона |

\*Базовий шлях у збірці задається у `vite.config.js` (`base`, зараз `/vector-analyzer-3ph/`). Якщо зміните репозиторій або ім’я Pages — оновіть `base` і посилання в маніфестах у `public/`.

### На Android (Chrome):
1. Відкрийте посилання на додаток (для мобільної інсталяції — сторінку `/mobile/`).
2. Натисніть на три крапки (меню) у верхньому правому куті.
3. Виберіть **"Додати на головний екран"** або **"Встановити додаток"**.

### На iPhone/iPad (Safari):
1. Відкрийте посилання у Safari.
2. Натисніть кнопку **"Поділитися"** (квадрат зі стрілкою вгору).
3. Прокрутіть вниз і виберіть **"Додати до початкового екрана"**.

### На комп'ютері (Chrome/Edge):
1. В адресному рядку браузера з'явиться іконка (монітор зі стрілкою).
2. Натисніть на неї та виберіть **"Встановити"**.

## 5. Створення звіту у PDF
У верній частині екрана є кнопка **«Експорт PDF»**. Звіт з діаграмою та таблицями **масштабується на один аркуш A4** (з полями). Для знімка DOM використовуються `html2canvas` та резервно `html-to-image`.

## 6. Основні гарячі клавіші та поради
- **Зміна кутів**: Ви можете перемикатися між режимом "Відносно UA" (класична полярна діаграма) та "Кут φ" (зручно для перевірки актів).
- **Фазна / лінійна напруга та схема (Y/Δ)**: для розрахунку P, Q, S використовується фазна напруга: при лінійній — для зірки U<sub>ф</sub> = U<sub>л</sub>/√3, для трикутника U<sub>ф</sub> = U<sub>л</sub>.
- **Кольори**: Жовтий - Фаза А, Зелений - Фаза B, Червоний - Фаза C.
- **Діагностика**: Якщо система бачить червону рамку в результатах — це критична помилка (наприклад, перевернута фаза).

## 7. Деплой на GitHub Pages

- Workflow: **`.github/workflows/deploy.yml`** — зазвичай запускається при **push** у гілку **`main`** або вручну (**Actions → Deploy static content to Pages → Run workflow**).
- Кроки збірки: `npm install` → `npm run build` → публікується вміст каталогу **`dist/`**.

### Обов’язково: джерело публікації

У репозиторії на GitHub: **Settings** → **Pages** → блок **Build and deployment** → поле **Source** має бути **GitHub Actions**.

Якщо обрано **Deploy from a branch** (наприклад, `master` / `/ (root)`), на сайт потрапляє **незібраний** проєкт: у `index.html` лишається посилання на `/src/main.jsx`, а браузер **не може** підвантажити React як у Vite-розробці — отримаєте **порожній екран**, помилки в консолі (`Failed to resolve module "react"` тощо), нібито «посилання не відкриваються».

**Що зробити:** перемкніть Source на **GitHub Actions**, збережіть, відкрийте **Actions**, запустіть workflow **Deploy static content to Pages** вручну (якщо потрібно), дочекайтесь зеленої галочки, через 1–2 хвилини оновіть сайт (краще з очищенням кешу або в режимі інкогніто).
